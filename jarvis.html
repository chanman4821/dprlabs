How we build · built on Jarvis

A small team,
amplified by a mesh.

Behind the studio is Jarvis — a group of specialised AI workers that plan the work, hand it to the right builder, grade each other’s output, and fix their own mistakes. It’s how a few senior people ship like a much larger team. We show it the only honest way: with a real run from our own logs.

The five roles

Plan, dispatch, build,
grade, and heal.

Each worker does one job well and checks the next. No single model is trusted with everything — the mesh catches its own mistakes.

  • Planner

    Breaks a goal into a small, ordered set of tasks with clear success criteria.

  • Dispatcher

    Routes each task to the worker best suited to it — and refuses work that isn’t ready.

  • Builder

    Does the actual work: writes the code, the docs, the configuration.

  • Grader

    Checks the result against the criteria and sends it back if it isn’t good enough.

  • Healer

    Catches failures, learns from them, and repairs the run without a human in the loop.
